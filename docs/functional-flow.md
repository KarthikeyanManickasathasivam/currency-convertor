## 1. Every-Request Filter Chain

Every HTTP request passes through this chain before reaching any controller. Order is fixed by Spring Security's filter registration.

```
Inbound request
    │
    ▼
CorrelationIdFilter          (HIGHEST_PRECEDENCE)
  - reads X-Correlation-ID header; generates UUID if absent
  - MDC.put("correlationId", id)  ← stamps every log line in this request
  - echoes ID back in response header
    │
    ▼
RateLimitFilter
  - extracts client IP from X-Forwarded-For (first entry) or remoteAddr
  - ConcurrentHashMap<IP, AtomicInteger> tracks hits per minute
  - if count > rate-limit.requests-per-minute (default 100): return 429 immediately
  - counters reset every 60 s via @Scheduled
    │
    ▼
JwtAuthFilter
  - reads Authorization: Bearer <token>
  - if no token → passes through (public endpoints proceed; protected ones later fail)
  - checks TokenBlacklistService.isBlacklisted(token) → Redis lookup
    if blacklisted → passes through (SecurityContext stays empty → 401 downstream)
  - JwtService.extractUsername(token) → parses RS256 JWT, extracts subject (email)
  - UserDetailsService.loadUserByUsername(email) → DB lookup → User entity
  - JwtService.isTokenValid(token, user) → verifies signature + expiry + email match
  - if valid: sets UsernamePasswordAuthenticationToken in SecurityContextHolder
    │
    ▼
Spring Security authorization check
  - /api/auth/**          → permitAll
  - /actuator/health      → permitAll
  - /api/rates/** GET     → authenticated
  - /api/admin/**         → hasRole('ADMIN')
  - everything else       → authenticated
    │
    ▼
Controller
```

## 2. Registration

**Endpoint:** `POST /api/auth/register`  
**Auth required:** None

```
Client → POST /api/auth/register
  body: { username, email, password }
    │
    ▼
AuthController.register()
    │
    ├─ Jakarta validation (@NotBlank, @Email, @Size) — 400 if fails
    │
    ▼
AuthService.register()
    │
    ├─ userRepository.existsByEmail(email)
    │    → if true: throw DuplicateResourceException → 409
    │
    ├─ userRepository.existsByUsername(username)
    │    → if true: throw DuplicateResourceException → 409
    │
    ├─ passwordEncoder.encode(password)
    │    BCrypt cost 12 (~250 ms intentionally slow)
    │
    ├─ User entity built:
    │    userId     = UUID (generated by DB)
    │    role       = USER (default)
    │    isActive   = true (default)
    │    createdAt  = set by @PrePersist
    │
    ├─ userRepository.save(user) → INSERT INTO users
    │
    └─ returns UserResponse (userId, username, email, role, isActive, createdAt)
    │
    ▼
AuthController.register()
    │
    ├─ logService.log("USER_REGISTERED", "AUTH", userId, clientIp,
    │                  {email}) → async INSERT INTO logs
    │
    └─ return 201 Created + UserResponse
```

## 3. Login — Standard MFA Path

**Endpoint:** `POST /api/auth/login`  
**Auth required:** None  
**Returns:** `202 Accepted` (no body) — signals OTP was sent

```
Client → POST /api/auth/login
  body: { email, password }
    │
    ▼
AuthController.login()
    │
    ▼
AuthService.login()
    │
    ├─ userRepository.findByEmail(email)
    │    → if not found: throw BadCredentialsException → 401
    │
    ├─ user.isActive() check
    │    → if false: throw BadCredentialsException → 401
    │
    ├─ passwordEncoder.matches(password, passwordHash)
    │    → if mismatch: throw BadCredentialsException → 401
    │
    ├─ email NOT in mfa.bypass-emails list → standard MFA path
    │
    ├─ otpService.generateAndStore(email)
    │    → generates 6-digit OTP
    │    → Redis SET key="otp:{email}" value=otp TTL=5min
    │
    ├─ emailService.sendOtp(email, otp)
    │    AWS: SesEmailService sends via SES SMTP
    │    Local: ConsoleEmailService prints formatted ASCII box to logs
    │
    └─ returns null (signals OTP sent, no token yet)
    │
    ▼
AuthController.login()
    │
    ├─ logService.log("LOGIN_ATTEMPT", "AUTH", null, clientIp, {email})
    │    → async INSERT INTO logs
    │
    └─ auth == null → return 202 Accepted (empty body)

─── User checks email, gets OTP ───

Client → POST /api/auth/mfa/verify
  body: { email, otp }
    │
    ▼
AuthController.verifyMfa()
    │
    ▼
AuthService.verifyMfa()
    │
    ├─ otpService.verify(email, otp)
    │    → Redis GET "otp:{email}" → compare → Redis DEL on match
    │    → if mismatch or expired: throw InvalidOtpException → 401
    │
    ├─ userRepository.findByEmail(email) → load User
    │
    ├─ jwtService.generateAccessToken(user)
    │    → RS256 JWT, subject=email, TTL=15min (900000ms)
    │    → no "type" claim → marks it as access token
    │
    └─ returns AuthResponse { accessToken, tokenType="Bearer", expiresIn, role }
    │
    ▼
AuthController.verifyMfa()
    │
    ├─ userRepository.findByEmail(email) → load User (second lookup, for refresh token)
    │
    ├─ authService.generateRefreshToken(user)
    │    → RS256 JWT, subject=email, TTL=7 days, claim type="refresh"
    │
    ├─ response.addCookie("refresh_token")
    │    HttpOnly=true, Secure=true (aws) / false (local)
    │    Path=/api/auth (scoped — only sent on auth endpoints)
    │    MaxAge = expiration/1000 * 48 seconds (7-day TTL)
    │
    ├─ logService.log("MFA_VERIFIED", "AUTH", null, clientIp, {email})
    │    → async INSERT INTO logs
    │
    └─ return 200 OK + AuthResponse
```

## 4. Login — MFA Bypass Path (POC)

Used for accounts listed in `app.mfa.bypass-emails`. Skips OTP entirely — issues token immediately at login.

```
Client → POST /api/auth/login
  body: { email, password }
    │
    ▼
AuthService.login()
    │
    ├─ credentials validated (same as standard path)
    │
    ├─ email IS in mfa.bypass-emails list → bypass path
    │    log.warn("MFA bypassed for {} (POC mode) — issuing token directly")
    │
    ├─ otpService.storeOtp(email, "000000")
    │    (dummy store — not used but keeps OTP service state consistent)
    │
    ├─ jwtService.generateAccessToken(user) → access token
    │
    └─ returns AuthResponse immediately (not null)
    │
    ▼
AuthController.login()
    │
    ├─ logService.log("LOGIN_ATTEMPT", ...) → async log
    │
    ├─ auth != null → generates refresh token + sets cookie
    │
    └─ return 200 OK + AuthResponse
         (no /mfa/verify step needed)
```

## 5. Token Refresh

**Endpoint:** `POST /api/auth/refresh`  
**Auth required:** `refresh_token` cookie (HttpOnly)

```
Client → POST /api/auth/refresh
  (refresh_token cookie sent automatically by browser on /api/auth path)
    │
    ▼
AuthController.refresh()
    │
    ├─ extractRefreshCookie(request) → reads cookie array, finds "refresh_token"
    │    → if not present: return 401 immediately
    │
    ▼
AuthService.refreshToken(refreshToken)
    │
    ├─ jwtService.isRefreshToken(token)
    │    → parses JWT, checks claim type == "refresh"
    │    → if not: throw InvalidOtpException → 401
    │
    ├─ jwtService.extractUsername(token) → gets email from subject
    │
    ├─ userRepository.findByEmail(email) → load User
    │    → if not found: throw ResourceNotFoundException → 404
    │
    ├─ jwtService.isTokenValid(token, user)
    │    → verifies signature, expiry, email match
    │    → if invalid: throw InvalidOtpException → 401
    │
    ├─ jwtService.generateAccessToken(user) → fresh 15-min access token
    │
    └─ returns AuthResponse { accessToken, tokenType, expiresIn, role }
    │
    ▼
AuthController.refresh()
    └─ return 200 OK + AuthResponse
         (refresh cookie is NOT rotated — same 7-day cookie until expiry)
```

## 6. Logout

**Endpoint:** `POST /api/auth/logout`  
**Auth required:** Bearer token

```
Client → POST /api/auth/logout
  header: Authorization: Bearer <accessToken>
    │
    ▼
JwtAuthFilter authenticates request (sets SecurityContext)
    │
    ▼
AuthController.logout()
    │
    ├─ extractBearerToken(request) → strips "Bearer " prefix
    │
    ├─ jwtService.extractExpiration(token)
    │    → remaining TTL = expiration.getTime() - System.currentTimeMillis()
    │
    ├─ tokenBlacklistService.blacklist(token, remainingTtl)
    │    → Redis SET key="blacklist:{token}" value="1" TTL=remainingTtl
    │    (token is now invalid for the rest of its natural lifetime)
    │
    ├─ clearRefreshCookie(response)
    │    → sets refresh_token cookie with MaxAge=0 → browser deletes it
    │
    ├─ logService.log("LOGOUT", "AUTH", null, clientIp, {email})
    │    → async INSERT INTO logs
    │
    └─ return 204 No Content
```

## 7. Fetch Exchange Rate

**Endpoint:** `GET /api/rates/{from}/{to}`  
**Auth required:** Any authenticated user

```
Client → GET /api/rates/USD/EUR
  header: Authorization: Bearer <token>
    │
    ▼
JwtAuthFilter → validates token
    │
    ▼
ExchangeRateController.getRate("USD", "EUR")
    │
    ▼
ExchangeRateService.getRateResponse("USD", "EUR")
    │
    ▼
ExchangeRateService.getRate("USD", "EUR")
  @Cacheable(value="rates", key="USD::EUR")
    │
    ├─ CACHE HIT (Redis key "rates::USD::EUR" exists, TTL 5 min)
    │    → returns cached BigDecimal immediately
    │    → log.debug("Rate USD/EUR served from DB")  ← note: label is "DB" even for cache hit
    │
    └─ CACHE MISS
         │
         ├─ rateRepository.findByFromCurrencyAndToCurrencyAndIsActiveTrue("USD","EUR")
         │    → if found in DB (manually set rate): return that, skip API call
         │
         └─ not in DB → call external API
              │
              ├─ primaryApiClient.getRate("USD","EUR")  [ExchangeRateHostClient]
              │    → @CircuitBreaker(name="rateApi", fallbackMethod="fallback")
              │    → @Retry(name="rateApi", maxAttempts=3, waitDuration=500ms)
              │    → WebClient call to exchangerate.host API
              │
              ├─ if primary fails → ExternalApiException thrown
              │    log.warn("Primary API failed for USD/EUR, trying fallback: ...")
              │    → fallbackApiClient.getRate("USD","EUR")  [AlphaVantageClient]
              │
              ├─ rate value obtained → saveOrUpdateRate("USD","EUR", rate, Source.API)
              │    → UPSERT into exchange_rates table
              │
              └─ @Cacheable populates Redis: "rates::USD::EUR" → rate, TTL 5 min
    │
    ▼
returns RateResponse { fromCurrency, toCurrency, rate, lastUpdated, source, isActive }
    │
    └─ return 200 OK + RateResponse
```

## 8. Currency Conversion — Below Threshold (Auto-Approved)

**Endpoint:** `POST /api/transactions`  
**Auth required:** Any authenticated user  
**Scenario:** Amount in USD equivalent is below the approval threshold (default $100)

```
Client → POST /api/transactions
  header: Authorization: Bearer <token>
  body: { fromCurrency: "GBP", toCurrency: "JPY", amount: 50.00 }
    │
    ▼
JwtAuthFilter → validates token, sets User in SecurityContext
    │
    ▼
TransactionController.convert(request, @AuthenticationPrincipal User user)
    │
    ▼
TransactionService.convert(request, user)
    │
    ├─ exchangeRateService.getRate("GBP", "JPY")
    │    → cache-aside pattern (as above) → returns rate e.g. 191.23
    │
    ├─ convertedAmount = 50.00 × 191.23 = 9561.50 (rounded HALF_UP, scale 2)
    │
    ├─ Threshold check:
    │    fromCurrency is not USD → need USD equivalent
    │    exchangeRateService.getRate("GBP","USD") → e.g. 1.27
    │    amountInUsd = 50.00 × 1.27 = 63.50
    │
    ├─ appSettingService.getApprovalThreshold()
    │    → SELECT * FROM app_settings WHERE setting_key = 'approval_threshold'
    │    → found: "100" → BigDecimal(100)
    │    → 63.50 < 100 → requiresApproval = false
    │
    ├─ status = APPROVED
    │
    ├─ Transaction entity built:
    │    transactionId  = UUID (DB generated)
    │    user           = authenticated User
    │    fromCurrency   = "GBP"
    │    toCurrency     = "JPY"
    │    amount         = 50.00
    │    convertedAmount= 9561.50
    │    rate           = 191.23 (immutable snapshot — updatable=false)
    │    transactionDate= set by @PrePersist
    │    status         = APPROVED
    │
    ├─ transactionRepository.save(tx) → INSERT INTO transactions
    │
    ├─ requiresApproval = false → no email sent, no approval log
    │
    └─ logService.log("TRANSACTION_CREATED", "TRANSACTION", userId, null,
                       {transactionId, status="APPROVED", amount, from, to})
         → async INSERT INTO logs
    │
    ▼
returns TransactionResponse (includes approvalThreshold in response so UI can show it)
    │
    └─ return 201 Created + TransactionResponse
```

## 9. Currency Conversion — Above Threshold (Pending Approval)

**Scenario:** Amount in USD equivalent meets or exceeds the approval threshold

```
Client → POST /api/transactions
  body: { fromCurrency: "USD", toCurrency: "INR", amount: 500.00 }
    │
    ▼
TransactionService.convert(request, user)
    │
    ├─ exchangeRateService.getRate("USD","INR") → e.g. 83.50
    ├─ convertedAmount = 500 × 83.50 = 41750.00
    │
    ├─ Threshold check:
    │    fromCurrency IS "USD" → amountInUsd = 500.00 (no conversion needed)
    │    approval threshold = 100
    │    500.00 >= 100 → requiresApproval = true
    │
    ├─ status = PENDING_APPROVAL
    │
    ├─ transactionRepository.save(tx) → INSERT INTO transactions (status=PENDING_APPROVAL)
    │
    ├─ emailService.sendApprovalRequiredNotification(
    │      to: adminEmail (from TRANSACTION_ADMIN_EMAIL env var),
    │      transactionId, userEmail, amount, fromCurrency, toCurrency)
    │    → admin receives email: "Transaction {id} requires approval"
    │
    ├─ log.info("Transaction {} requires admin approval. Amount: 500 USD", txId)
    │
    └─ logService.log("TRANSACTION_CREATED", "TRANSACTION", userId, null,
                       {transactionId, status="PENDING_APPROVAL", amount, from, to})
         → async INSERT INTO logs
    │
    └─ return 201 Created + TransactionResponse (status="PENDING_APPROVAL")

─── User sees status "PENDING_APPROVAL" in their transaction history ───
─── Admin takes action (see flows 12 and 13) ───
```

## 11. Admin: User Management

### List all users
**Endpoint:** `GET /api/admin/users?page=0&size=20`

```
AdminController.listUsers(pageable)
  → userService.getAllUsers(pageable)
  → userRepository.findAllByOrderByCreatedAtDesc(pageable)
  → returns Page<UserResponse> (userId, username, email, role, isActive, createdAt, updatedAt)
  → return 200 OK
```

### Get one user
**Endpoint:** `GET /api/admin/users/{id}`

```
AdminController.getUser(id)
  → userService.getUserById(id)
  → userRepository.findById(id) → if absent: 404
  → return 200 OK + UserResponse
```

### Deactivate a user
**Endpoint:** `DELETE /api/admin/users/{id}`

```
AdminController.deactivateUser(id)
  → userService.deactivateUser(id)
  → userRepository.findById(id) → if absent: 404
  → user.setActive(false)
  → userRepository.save(user)   → UPDATE users SET is_active=false WHERE user_id=?
  → return 204 No Content

Note: deactivated users cannot login — isAccountNonLocked() returns false on User entity,
causing Spring Security to reject authentication with DisabledException → 403.
```

## 12. Admin: Approve a Transaction

**Endpoint:** `POST /api/admin/transactions/{id}/approve`  
**Auth required:** ADMIN role

```
Client → POST /api/admin/transactions/{txId}/approve
  header: Authorization: Bearer <admin-token>
    │
    ▼
AdminController.approve(txId, @AuthenticationPrincipal User admin)
    │
    ▼
TransactionService.approve(txId, admin.getUserId())
    │
    ├─ transactionRepository.findById(txId)
    │    → if absent: throw TransactionNotFoundException → 404
    │
    ├─ tx.getStatus() != PENDING_APPROVAL
    │    → throw IllegalStateException → 500
    │    (prevents double-approval)
    │
    ├─ userRepository.findById(adminId) → load admin User entity
    │
    ├─ tx.setStatus(APPROVED)
    ├─ tx.setApprovedBy(admin)
    ├─ tx.setApprovalDate(LocalDateTime.now())
    ├─ transactionRepository.save(tx)
    │    → UPDATE transactions SET status='APPROVED', approved_by=?, approval_date=? WHERE ...
    │
    ├─ emailService.sendApprovalNotification(
    │      to: tx.getUser().getEmail(),
    │      transactionId, convertedAmount, toCurrency)
    │    → user receives: "Your transaction has been approved"
    │
    └─ logService.log("TRANSACTION_APPROVED", "TRANSACTION", adminId, null,
                       {transactionId})
         → async INSERT INTO logs
    │
    └─ return 200 OK + TransactionResponse (status="APPROVED", approvedBy=adminId, approvalDate)
```

## 13. Admin: Reject a Transaction

**Endpoint:** `POST /api/admin/transactions/{id}/reject`  
**Auth required:** ADMIN role

```
Client → POST /api/admin/transactions/{txId}/reject
  body: { "reason": "Exceeds daily limit per policy" }   (optional)
    │
    ▼
TransactionService.reject(txId, admin.getUserId(), reason)
    │

    │

    │    → UPDATE transactions SET status='REJECTED', approved_by=?, approval_date=? WHERE ...
    │

    │      to: tx.getUser().getEmail(), transactionId, reason)
    │    → user receives: "Your transaction has been rejected" + reason if provided
    │
    └─ logService.log("TRANSACTION_REJECTED", "TRANSACTION", adminId, null,
                       {transactionId, reason})
         → async INSERT INTO logs
    │
    └─ return 200 OK + TransactionResponse (status="REJECTED")
```

## 14. Admin: Manage Exchange Rates

### Create a manual rate
**Endpoint:** `POST /api/rates/admin`  
**Auth required:** ADMIN role  
**Purpose:** Override or seed a rate that won't hit the external API

```
AdminController (via ExchangeRateController.createRate())
    │
    ▼
ExchangeRateService.createRate(request)
  @CacheEvict(value="rates", key for this pair)
    │
    ├─ rateRepository.existsByFromCurrencyAndToCurrency(from, to)
    │    → if exists: throw DuplicateResourceException → 409
    │
    ├─ ExchangeRate entity: source=MANUAL, isActive=true
    ├─ rateRepository.save(rate) → INSERT INTO exchange_rates
    ├─ cache entry for this pair evicted from Redis
    └─ return 201 Created + RateResponse
```

### Update a rate
**Endpoint:** `PUT /api/rates/admin/{id}`

```
ExchangeRateService.updateRate(id, request)
  @CacheEvict(value="rates", allEntries=true)   ← evicts ALL rate cache entries
    │
    ├─ rateRepository.findById(id) → if absent: 404
    ├─ rate.setRate(newRate), rate.setSource(MANUAL)
    ├─ rateRepository.save(rate) → UPDATE exchange_rates
    └─ return 200 OK + RateResponse
```

### Soft-delete a rate
**Endpoint:** `DELETE /api/rates/admin/{id}`

```
ExchangeRateService.deleteRate(id)
  @CacheEvict(value="rates", allEntries=true)
    │
    ├─ rateRepository.findById(id) → if absent: 404
    ├─ rate.setActive(false)
    ├─ rateRepository.save(rate) → UPDATE exchange_rates SET is_active=false
    │    (row is kept for audit; isActive=false excludes it from getRate() lookups)
    └─ return 204 No Content
```

## 15. Admin: Approval Threshold Settings

### Get current threshold
**Endpoint:** `GET /api/admin/settings/approval-threshold`

```
AdminController.getApprovalThreshold()
    │
    ├─ appSettingRepository.findById("approval_threshold")
    │    → if present: returns setting with value, updatedAt, updatedBy user
    │    → if absent: threshold = defaultThreshold from application.yml (100)
    │
    └─ return 200 OK + ApprovalThresholdResponse { threshold, updatedAt, updatedBy }
```

### Update threshold
**Endpoint:** `PUT /api/admin/settings/approval-threshold`

```
Client → PUT /api/admin/settings/approval-threshold
  body: { "threshold": 250.00 }
    │
    ▼
AdminController.updateApprovalThreshold(request, @AuthenticationPrincipal User admin)
    │
    ▼
AppSettingService.updateApprovalThreshold(250.00, admin)
    │
    ├─ appSettingRepository.findById("approval_threshold")
    │    → if exists: update existing record
    │    → if absent: create new AppSetting with key="approval_threshold"
    │
    ├─ setting.setValue("250.00")
    ├─ setting.setUpdatedAt(LocalDateTime.now())
    ├─ setting.setUpdatedBy(admin)
    ├─ appSettingRepository.save(setting) → UPSERT INTO app_settings
    │
    └─ log.info("Approval threshold updated to 250.00 by admin {}", adminId)
    │
    ▼
AdminController re-reads the saved setting for the response
    └─ return 200 OK + ApprovalThresholdResponse { threshold=250.00, updatedAt, updatedBy=adminId }

Effect: all subsequent conversion requests will use 250.00 as the threshold.
AppSettingService.getApprovalThreshold() always reads from DB — no caching,
so the new value takes effect immediately on the next conversion.
```

## 16. Admin: Audit Logs

### All logs (paginated)
**Endpoint:** `GET /api/admin/logs?page=0&size=50`

```
AdminController.getLogs(pageable)
  → logRepository.findAllByOrderByTimestampDesc(pageable)
  → return 200 OK + Page<LogResponse>
       { logId, event, eventType, timestamp, userId, ipAddress, details (JSON) }
```

### Logs by event type
**Endpoint:** `GET /api/admin/logs/type/{eventType}`  
Example: `GET /api/admin/logs/type/TRANSACTION`

```
AdminController.getLogsByType("TRANSACTION", pageable)
  → logRepository.findByEventTypeOrderByTimestampDesc("TRANSACTION", pageable)
  → return 200 OK + Page<LogResponse>
```

**Event types in use:**

| eventType | event values |
|---|---|
| AUTH | USER_REGISTERED, LOGIN_ATTEMPT, MFA_VERIFIED, LOGOUT |
| TRANSACTION | TRANSACTION_CREATED, TRANSACTION_APPROVED, TRANSACTION_REJECTED |

## 17. Error Paths

All errors return a standard `ErrorResponse`:
```json
{
  "status": 401,
  "error": "Unauthorized",
  "message": "Invalid email or password",
  "path": "/api/auth/login",
  "timestamp": "2026-06-20T12:34:56",
  "fieldErrors": null
}
```

| Scenario | Exception | HTTP | Logged |
|---|---|---|---|
| Duplicate email/username on register | `DuplicateResourceException` | 409 | WARN at path + message |
| Wrong password / unknown email | `BadCredentialsException` | 401 | WARN at path (no credential detail) |
| Wrong or expired OTP | `InvalidOtpException` | 401 | WARN at path only |
| Disabled account | `DisabledException` | 403 | WARN at path |
| Non-admin hits /api/admin/** | `AccessDeniedException` | 403 | WARN at path |
| Resource not found | `ResourceNotFoundException` | 404 | WARN at path + message |
| Transaction not found | `TransactionNotFoundException` | 404 | WARN at path + message |
| Bean validation fails | `MethodArgumentNotValidException` | 400 | WARN with field→message map |
| External rate API down (both primary + fallback fail) | `ExternalApiException` | 503 | ERROR with stack trace |
| Anything else | `Exception` | 500 | ERROR with stack trace |
| Rate limit exceeded | — (handled in filter) | 429 | WARN with client IP |

Stack traces are only logged for 503 (external API) and 500 (unexpected) — all 4xx paths log at WARN without stack traces to avoid log noise.
