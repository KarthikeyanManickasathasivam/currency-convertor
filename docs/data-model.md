# Data Model & Database Design

## Entity: users

| Field | Java Type | JPA Annotations | Notes |
|-------|-----------|-----------------|-------|
| userId | UUID | `@Id @GeneratedValue(strategy = UUID)` `@Column(updatable = false)` | UUID PK; generated by `gen_random_uuid()` in DB |
| username | String | `@Column(unique = true, nullable = false, length = 50)` | Display name; login is by email |
| email | String | `@Column(unique = true, nullable = false, length = 255)` | Used as `UserDetails.getUsername()` — JWT subject and `loadUserByUsername` key; required for MFA OTP and SES notifications |
| passwordHash | String | `@Column(nullable = false)` | BCrypt hash (cost 12); never returned in API responses |
| role | Role (enum) | `@Enumerated(EnumType.STRING)` `@Column(nullable = false, length = 20)` | Default `USER`; embedded in JWT claims |
| isActive | Boolean | `@Column(nullable = false)` | Default `true`. Soft-delete flag. `isAccountNonLocked()` and `isEnabled()` delegate to this. |
| createdAt | LocalDateTime | `@Column(nullable = false, updatable = false)` | Set via `@PrePersist` |
| updatedAt | LocalDateTime | `@Column(nullable = false)` | Set via `@PrePersist` and `@PreUpdate` |

**Notes:**
- Implements `UserDetails`; `getUsername()` returns `email`, `getDisplayUsername()` returns `username`
- Auditing uses `@PrePersist` / `@PreUpdate` lifecycle hooks — not Spring Data `@CreatedDate`/`@LastModifiedDate`

## Entity: exchange_rates

| Field | Java Type | JPA Annotations | Notes |
|-------|-----------|-----------------|-------|
| id | Long | `@Id @GeneratedValue(strategy = IDENTITY)` `@Column(updatable = false)` | BIGSERIAL PK |
| fromCurrency | String | `@Column(nullable = false, length = 3)` | ISO 4217 code (e.g., `USD`) |
| toCurrency | String | `@Column(nullable = false, length = 3)` | ISO 4217 code; composite unique with `fromCurrency` enforced by DB constraint |
| rate | BigDecimal | `@Column(nullable = false, precision = 18, scale = 8)` | 8 decimal places for forex precision; DB CHECK enforces `rate > 0` |
| lastUpdated | LocalDateTime | `@Column(nullable = false)` | Set via `@PrePersist` and `@PreUpdate` |
| source | Source (enum) | `@Enumerated(EnumType.STRING)` `@Column(nullable = false, length = 10)` | Default `API` via `@Builder.Default` |
| isActive | Boolean | `@Column(nullable = false)` | Default `true` via `@Builder.Default`. Soft-delete flag. |

**Notes:**
- The composite unique constraint `(from_currency, to_currency)` is enforced at the DB level only (Flyway DDL); no `@UniqueConstraint` on the entity
- `lastUpdated` is auto-set on every save — not only on creation

## Entity: transactions

| Field | Java Type | JPA Annotations | Notes |
|-------|-----------|-----------------|-------|
| transactionId | UUID | `@Id @GeneratedValue(strategy = UUID)` `@Column(updatable = false)` | UUID PK; generated by `gen_random_uuid()` in DB |
| user | User | `@ManyToOne(fetch = LAZY)` `@JoinColumn(name = "user_id", nullable = false)` | FK to users; LAZY to avoid N+1 |
| fromCurrency | String | `@Column(nullable = false, length = 3)` | Source currency |
| toCurrency | String | `@Column(nullable = false, length = 3)` | Target currency |
| amount | BigDecimal | `@Column(nullable = false, precision = 18, scale = 2)` | Original amount; DB CHECK enforces `amount > 0` |
| convertedAmount | BigDecimal | `@Column(nullable = false, precision = 18, scale = 2)` | Converted result |
| rate | BigDecimal | `@Column(nullable = false, precision = 18, scale = 8, updatable = false)` | Immutable rate snapshot at conversion time |
| transactionDate | LocalDateTime | `@Column(nullable = false, updatable = false)` | Set via `@PrePersist` |
| status | TransactionStatus (enum) | `@Enumerated(EnumType.STRING)` `@Column(nullable = false, length = 20)` | Default `APPROVED` via `@Builder.Default`. Transitions to `PENDING_APPROVAL` when amount ≥ threshold. |
| approvedBy | User | `@ManyToOne(fetch = LAZY)` `@JoinColumn(name = "approved_by")` | Nullable; admin who approved or rejected; `NULL` for auto-approved transactions |
| approvalDate | LocalDateTime | `@Column` | Nullable; timestamp of admin action |

**Notes:**
- `transactionDate` uses `@PrePersist` lifecycle hook, not Spring Data auditing
- `rate` is `updatable = false` — the rate snapshot is frozen at creation and cannot be changed

## Entity: logs

| Field | Java Type | JPA Annotations | Notes |
|-------|-----------|-----------------|-------|
| logId | Long | `@Id @GeneratedValue(strategy = IDENTITY)` `@Column(updatable = false)` | BIGSERIAL PK |
| event | String | `@Column(nullable = false, length = 100)` | Short event description |
| eventType | String | `@Column(nullable = false, length = 50)` | `LOGIN`, `LOGOUT`, `CONVERSION`, `ADMIN_ACTION`, `API_CALL`, `ERROR` |
| timestamp | LocalDateTime | `@Column(nullable = false, updatable = false)` | Set via `@PrePersist` |
| userId | UUID | `@Column` | Nullable plain column (no JPA relationship). `NULL` for system-level events. DB FK with `ON DELETE SET NULL`. |
| ipAddress | String | `@Column(length = 45)` | IPv4 or IPv6 |
| details | Map\<String, Object\> | `@JdbcTypeCode(SqlTypes.JSON)` `@Column` | Stored as JSONB in PostgreSQL. GIN-indexed. |

**Notes:**
- `details` is mapped as `Map<String, Object>` with Hibernate's JSON type — NOT a plain `String`
- `userId` is a raw `UUID` column, not a `@ManyToOne` — this allows DB `ON DELETE SET NULL` without a JPA cascade
- `timestamp` uses `@PrePersist`, not Spring Data auditing

## Entity: app_settings

| Field | Java Type | JPA Annotations | Notes |
|-------|-----------|-----------------|-------|
| key | String | `@Id @Column(name = "setting_key", nullable = false, length = 100)` | Natural string PK (e.g., `approval_threshold`) |

| updatedAt | LocalDateTime | `@Column` | Nullable; last update timestamp |

## Flyway Migrations

### V1__create_users.sql
```sql
CREATE TABLE users (
    user_id       UUID         NOT NULL DEFAULT gen_random_uuid(),
    username      VARCHAR(50)  NOT NULL,
    email         VARCHAR(255) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role          VARCHAR(20)  NOT NULL DEFAULT 'USER',
    is_active     BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMP    NOT NULL DEFAULT NOW(),

    CONSTRAINT pk_users          PRIMARY KEY (user_id),
    CONSTRAINT uq_users_username UNIQUE (username),
    CONSTRAINT uq_users_email    UNIQUE (email),
    CONSTRAINT chk_users_role    CHECK (role IN ('USER', 'ADMIN'))
);

CREATE INDEX idx_users_email     ON users (email);
CREATE INDEX idx_users_username  ON users (username);
CREATE INDEX idx_users_is_active ON users (is_active);
```

### V2__create_exchange_rates.sql
```sql
CREATE TABLE exchange_rates (
    id            BIGSERIAL      NOT NULL,
    from_currency VARCHAR(3)     NOT NULL,
    to_currency   VARCHAR(3)     NOT NULL,
    rate          DECIMAL(18, 8) NOT NULL,
    last_updated  TIMESTAMP      NOT NULL DEFAULT NOW(),
    source        VARCHAR(10)    NOT NULL DEFAULT 'API',
    is_active     BOOLEAN        NOT NULL DEFAULT TRUE,

    CONSTRAINT pk_exchange_rates       PRIMARY KEY (id),
    CONSTRAINT uq_exchange_rates_pair  UNIQUE (from_currency, to_currency),
    CONSTRAINT chk_exchange_rates_source   CHECK (source IN ('API', 'MANUAL')),
    CONSTRAINT chk_exchange_rates_positive CHECK (rate > 0)
);

CREATE INDEX idx_exchange_rates_pair         ON exchange_rates (from_currency, to_currency);
CREATE INDEX idx_exchange_rates_active       ON exchange_rates (is_active);
CREATE INDEX idx_exchange_rates_last_updated ON exchange_rates (last_updated);
```

### V3__create_transactions.sql
```sql
CREATE TABLE transactions (
    transaction_id   UUID           NOT NULL DEFAULT gen_random_uuid(),
    user_id          UUID           NOT NULL,
    from_currency    VARCHAR(3)     NOT NULL,
    to_currency      VARCHAR(3)     NOT NULL,
    amount           DECIMAL(18, 2) NOT NULL,
    converted_amount DECIMAL(18, 2) NOT NULL,
    rate             DECIMAL(18, 8) NOT NULL,
    transaction_date TIMESTAMP      NOT NULL DEFAULT NOW(),
    status           VARCHAR(20)    NOT NULL DEFAULT 'APPROVED',
    approved_by      UUID,
    approval_date    TIMESTAMP,

    CONSTRAINT pk_transactions          PRIMARY KEY (transaction_id),
    CONSTRAINT fk_transactions_user     FOREIGN KEY (user_id)     REFERENCES users (user_id),
    CONSTRAINT fk_transactions_approver FOREIGN KEY (approved_by) REFERENCES users (user_id),
    CONSTRAINT chk_transactions_status  CHECK (status IN ('PENDING_APPROVAL', 'APPROVED', 'REJECTED')),
    CONSTRAINT chk_transactions_amount_positive CHECK (amount > 0)
);

CREATE INDEX idx_transactions_user_id ON transactions (user_id);
CREATE INDEX idx_transactions_status  ON transactions (status);
CREATE INDEX idx_transactions_date    ON transactions (transaction_date DESC);
```

### V4__create_logs.sql
```sql
CREATE TABLE logs (
    log_id     BIGSERIAL    NOT NULL,
    event      VARCHAR(100) NOT NULL,
    event_type VARCHAR(50)  NOT NULL,

CREATE INDEX idx_logs_user_id     ON logs (user_id);
CREATE INDEX idx_logs_event_type  ON logs (event_type);
CREATE INDEX idx_logs_details_gin ON logs USING GIN (details);
```

## Repositories

### UserRepository — `JpaRepository<User, UUID>`
| Method | Purpose |
|--------|---------|
| `findByEmail(String)` | Primary lookup — `UserDetailsService.loadUserByUsername` |
| `findByUsername(String)` | Username lookup |
| `existsByEmail(String)` | Duplicate check on registration |
| `existsByUsername(String)` | Duplicate check on registration |
| `findAllByOrderByCreatedAtDesc(Pageable)` | Admin user list |
| `countActiveByRole(Role)` *(JPQL)* | Dashboard stats per role |
| `countActive()` *(JPQL)* | Dashboard total active users |

### ExchangeRateRepository — `JpaRepository<ExchangeRate, Long>`
| Method | Purpose |
|--------|---------|
| `findByFromCurrencyAndToCurrencyAndIsActiveTrue(String, String)` | Cache-miss lookup; used in `getRate()` and `saveOrUpdateRate()` |
| `findAllByIsActiveTrueOrderByFromCurrencyAsc()` | Active rate list for users |
| `findAllByOrderByFromCurrencyAsc(Pageable)` | Admin paginated rate list (active + inactive) |
| `existsByFromCurrencyAndToCurrency(String, String)` | Duplicate check in `createRate()` |
| `findDistinctActiveCurrencies()` *(JPQL)* | Populates currency dropdowns |

### TransactionRepository — `JpaRepository<Transaction, UUID>`
| Method | Purpose |
|--------|---------|
| `findByUserUserIdOrderByTransactionDateDesc(UUID, Pageable)` | User's own transaction history |
| `findAllByOrderByTransactionDateDesc(Pageable)` | Admin all-transactions view |
| `findByStatusOrderByTransactionDateDesc(TransactionStatus)` | Pending approval queue |
| `countByStatus(TransactionStatus)` | Dashboard status counts |
| `countAll()` *(JPQL)* | Dashboard total count |
| `findTopFromCurrencies(Pageable)` *(JPQL aggregate)* | Top source currencies for dashboard |

### LogRepository — `JpaRepository<Log, Long>`
| Method | Purpose |
|--------|---------|
| `findAllByOrderByTimestampDesc(Pageable)` | Admin log viewer |
| `findByUserIdOrderByTimestampDesc(UUID, Pageable)` | Per-user audit trail |
| `findByEventTypeOrderByTimestampDesc(String, Pageable)` | Filter by event type |
| `findByTimestampBetweenOrderByTimestampDesc(LocalDateTime, LocalDateTime, Pageable)` | Date-range log query |

### AppSettingRepository — `JpaRepository<AppSetting, String>`
No custom queries. Uses inherited `findById(String key)` and `save()`.

## Redis Key Patterns

| Key Pattern | Value | TTL | Notes |
|-------------|-------|-----|-------|
| `rates::{FROM}::{TO}` | Cached `BigDecimal` rate | 5 min | `@Cacheable(value="rates", key="#fromCurrency + '::' + #toCurrency")` — Spring Cache via Lettuce |
| `token:blacklist:{jti}` | `"revoked"` | Matches token expiry | Written on logout; checked in `JwtAuthFilter` |
| `mfa:otp:{userId}` | JSON `{"otp":"…","attempts":0}` | 5 min | 6-digit OTP; max 3 attempts; cleared on successful verify |
| `ratelimit:{ip}` | Counter | 1 min | Managed by `RateLimitFilter` |

**Cache key note:** The Spring Cache key uses `::` as separator (e.g., `rates::USD::EUR`), matching the `@CacheEvict` key expressions in `ExchangeRateService`.

## Database Connection Config (application.yml)

```yaml
spring:
  datasource:
    url: jdbc:postgresql://${DB_HOST:localhost}:5432/${DB_NAME:exchange_db}
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      idle-timeout: 30000
      connection-timeout: 10000
  jpa:
    hibernate:
      ddl-auto: validate   # Flyway owns the schema; Hibernate only validates
    properties:
      hibernate:
        default_schema: public
  flyway:
    enabled: true
    locations: classpath:db/migration
```
